# Лабораторная работа №6 — Настройка сети в Cisco Packet Tracer

По таблице подсетей:
| Подсеть | Хостов | Маска | Сеть | Первый IP | Последний IP |
|--------|--------|-------|------|-----------|--------------|
| E | 44 | /26 | 68.76.115.0 | 68.76.115.1 | 68.76.115.62 |
| B | 25 | /27 | 68.76.115.64 | 68.76.115.65 | 68.76.115.94 |
| C | 25 | /27 | 68.76.115.96 | 68.76.115.97 | 68.76.115.126 |
| A | 16 | /27 | 68.76.115.128 | 68.76.115.129 | 68.76.115.158 |
| D | 10 | /28 | 68.76.115.160 | 68.76.115.161 | 68.76.115.174 |

**Рекомендуемое назначение:**  
- **E** — основная LAN (клиенты, DHCP).  
- **B, C** — отдельные сегменты (например, отделы).  
- **A** — серверный сегмент (DNS, HTTP/FTP, почта, поиск).  
- **D** — сегмент для серверов или DMZ (при необходимости).  

---

## Сколько чего добавить и как соединить

### Список оборудования (минимум)

| Количество | Устройство | Тип в Packet Tracer |
|------------|------------|----------------------|
| 1 | Маршрутизатор | Router (например 2911 или 1941), с 6 интерфейсами LAN |
| 5 | Коммутаторы | Switch (2960 или 2950) — по одному на каждую подсеть E, B, C, A, D |
| 0–2 | Хабы | Hub (при необходимости по заданию — обычно не нужны, если есть свитчи) |
| 1 | «Интернет» | Cloud (Cloud-PT) или второй роутер — для NAT и выхода наружу |
| 4 | Серверы | Server-PT (DNS; Web+FTP; Mail; Search — можно объединить Web и FTP на одном) |
| 3–5 | ПК клиентов | PC (по одному в подсетях E, B, C для проверки DHCP и доступа) |

**Итого по минимуму:** 1 роутер, 5 свитчей, 1 Cloud (или 2-й роутер), 4 сервера, 3–5 ПК. Хабы — только если явно требуются в задании.

---

### Схема подключений

```
                    [Cloud или Router-ISP]   «Интернет»
                              |
                         (кабель)
                              |
    ┌─────────────────────────┼─────────────────────────┐
    |                         |                         |
    |  R1 (маршрутизатор)     |                         |
    |  • Gi0/0 — подсеть E    |  • Gi0/5 — в Интернет   |
    |  • Gi0/1 — подсеть B    |                         |
    |  • Gi0/2 — подсеть C    |                         |
    |  • Gi0/3 — подсеть A    |                         |
    |  • Gi0/4 — подсеть D    |                         |
    └───┬─────┬─────┬─────┬─────┬──────────────────────┘
        |     |     |     |     |
    [SW-E] [SW-B] [SW-C] [SW-A] [SW-D]   ← по одному свитчу на подсеть
        |     |     |     |     |
     ПК +   ПК    ПК   серверы  (пусто или
    (клиенты)         (DNS,     серверы)
                      Web/FTP,
                      Mail,
                      Search)
```

---

### Кто с чем соединять (пошагово)

1. **Роутер R1**
   - **GigabitEthernet0/0** ↔ свитч **SW-E** (подсеть E).
   - **GigabitEthernet0/1** ↔ свитч **SW-B** (подсеть B).
   - **GigabitEthernet0/2** ↔ свитч **SW-C** (подсеть C).
   - **GigabitEthernet0/3** ↔ свитч **SW-A** (подсеть A, серверы).
   - **GigabitEthernet0/4** ↔ свитч **SW-D** (подсеть D).
   - **GigabitEthernet0/5** (или свободный порт) ↔ **Cloud** или второй роутер («Интернет»).

2. **Свитчи**
   - К **SW-E**: роутер (один порт) + ПК клиентов (остальные порты). Кабель: Copper Straight-Through (зелёный).
   - К **SW-B**: роутер + 1–2 ПК.
   - К **SW-C**: роутер + 1–2 ПК.
   - К **SW-A**: роутер + все 4 сервера (DNS, Web/FTP, Mail, Search). Каждый сервер — своим портом.
   - К **SW-D**: роутер; при необходимости — дополнительные серверы или ПК.

3. **Серверы** (все вешаются на SW-A)
   - Подключить к свободным портам SW-A. На каждом сервере в настройках интерфейса указать: IP из подсети A (68.76.115.128/27), шлюз 68.76.115.129.

4. **ПК**
   - Подключить к SW-E, SW-B, SW-C. На ПК: либо DHCP, либо статический IP из своей подсети; шлюз — IP роутера в этой подсети (68.76.115.1 для E, 68.76.115.65 для B, 68.76.115.97 для C).

5. **Cloud (Интернет)**
   - В Packet Tracer: добавить Cloud, настроить на одном из портов адрес «провайдера» (например 203.0.113.2/24). Роутер R1 на Gi0/5 — 203.0.113.1. Между R1 и Cloud — кабель Copper Straight-Through.

---

### Краткая таблица «что куда»

| Устройство | Подключено к | Кабель |
|------------|--------------|--------|
| R1 Gi0/0 | SW-E Fa0/1 (или любой порт) | Straight-Through |
| R1 Gi0/1 | SW-B Fa0/1 | Straight-Through |
| R1 Gi0/2 | SW-C Fa0/1 | Straight-Through |
| R1 Gi0/3 | SW-A Fa0/1 | Straight-Through |
| R1 Gi0/4 | SW-D Fa0/1 | Straight-Through |
| R1 Gi0/5 | Cloud или Router-ISP | Straight-Through |
| ПК (подсеть E) | SW-E | Straight-Through |
| ПК (подсеть B, C) | SW-B, SW-C | Straight-Through |
| Серверы (4 шт.) | SW-A | Straight-Through |

После сборки схемы настраивайте роутер, серверы и NAT по разделам ниже.

## 1. Маршрутизатор (Router)

### 1.1. Включение и базовые настройки

```
enable
configure terminal
hostname R1
no ip domain-lookup
line console 0
 logging synchronous
 exit
exit
write memory
```

### 1.2. Назначение IP на интерфейсы

Подставьте свои типы интерфейсов (например, GigabitEthernet0/0, GigabitEthernet0/1) в соответствии со схемой.

**Пример: подсеть E на fa0/0, подсеть A (серверы) на fa0/1, «Интернет» на fa0/2.**

```
configure terminal

! Подсеть E (68.76.115.0/26) — клиентская LAN
interface GigabitEthernet0/0
 ip address 68.76.115.1 255.255.255.192
 no shutdown
 exit

! Подсеть B (68.76.115.64/27)
interface GigabitEthernet0/1
 ip address 68.76.115.65 255.255.255.224
 no shutdown
 exit

! Подсеть C (68.76.115.96/27)
interface GigabitEthernet0/2
 ip address 68.76.115.97 255.255.255.224
 no shutdown
 exit

! Подсеть A (68.76.115.128/27) — серверы
interface GigabitEthernet0/3
 ip address 68.76.115.129 255.255.255.224
 no shutdown
 exit

! Подсеть D (68.76.115.160/28)
interface GigabitEthernet0/4
 ip address 68.76.115.161 255.255.255.240
 no shutdown
 exit

! Интерфейс в «Интернет» (маска и адрес — по заданию ЛР6)
interface GigabitEthernet0/5
 ip address 203.0.113.1 255.255.255.0
 no shutdown
 exit

exit
write memory
```

### 1.3. Маршрут по умолчанию (если «Интернет» — через один выход)

```
ip route 0.0.0.0 0.0.0.0 203.0.113.2
```

Адрес шлюза (203.0.113.2) замените на адрес «провайдера» по вашей схеме.

## 2. DHCP-сервер

### Вариант A: DHCP на маршрутизаторе (для подсети E)

```
configure terminal
ip dhcp excluded-address 68.76.115.1 68.76.115.10

ip dhcp pool LAN-E
 network 68.76.115.0 255.255.255.192
 default-router 68.76.115.1
 dns-server 68.76.115.130
 exit

exit
write memory
```

При необходимости создайте пулы для B и C по тому же принципу (default-router = IP маршрутизатора в этой подсети, dns-server = IP вашего DNS).

### Вариант B: DHCP через сервер в Packet Tracer

1. Добавьте сервер (Server-PT).
2. Назначьте ему статический IP из подсети, где сидят клиенты (например, 68.76.115.2 в подсети E), маску и шлюз (68.76.115.1).
3. **Services → DHCP:**
   - **Service: On**
   - **Pool Name:** LAN-E  
   - **Default Gateway:** 68.76.115.1  
   - **DNS Server:** 68.76.115.130 (IP DNS-сервера)  
   - **Start IP:** 68.76.115.11  
   - **Subnet Mask:** 255.255.255.192  
   - **Max Users:** по заданию (например, 50).

---

## 3. DNS-сервер

1. Добавьте сервер (Server-PT) в подсеть A (или D).
2. **IP:** 68.76.115.130, маска 255.255.255.224, шлюз 68.76.115.129.
3. **Services → DNS:**
   - **Service: On**
   - Добавьте записи, например:

| Name | Address |
|------|---------|
| www.company.com | 68.76.115.131 |
| ftp.company.com | 68.76.115.131 |
| mail.company.com | 68.76.115.132 |
| search.company.com | 68.76.115.133 |

Адреса замените на фактические IP ваших HTTP/FTP, почтового и поискового серверов.

---

## 4. HTTP и FTP сервер (один сервер может отдавать оба)

1. Сервер в подсети A: например **68.76.115.131**, маска /27, шлюз 68.76.115.129.
2. **Services → HTTP:** включите **HTTP** (и при необходимости **HTTPS**).
3. **Services → FTP:** включите **FTP**. При необходимости задайте пользователя/пароль в FTP.
4. В DNS (см. выше) укажите для этого IP имена www и ftp.

---

## 5. Почтовый сервер (Mail)

1. Сервер в подсети A: например **68.76.115.132**, шлюз 68.76.115.129.
2. **Services → EMAIL:**
   - **Service: On**
   - **Domain Name:** company.com (или по заданию)
   - **User** / **Password** — создайте учётные записи для проверки.
3. В DNS добавьте запись **mail.company.com** → 68.76.115.132.

---

## 6. Поисковый сервер (Search)

1. В Packet Tracer поиск часто реализуют как веб-сервер с поиском или как отдельный сервис на сервере.
2. Сервер: например **68.76.115.133**, шлюз 68.76.115.129.
3. **Services:** включите **HTTP** (и при необходимости **Search**/доп. сервисы, если есть в вашей версии PT).
4. В DNS: **search.company.com** → 68.76.115.133.

---

## 7. NAT (на маршрутизаторе, выходящем в «Интернет»)

Предполагается: внутренние сети 68.76.115.0/24 (все ваши подсети), внешний интерфейс — GigabitEthernet0/5 (203.0.113.1).

```
configure terminal

! Список ACL для «внутренних» сетей (разрешаем то, что будем пускать в NAT)
access-list 1 permit 68.76.115.0 0.0.0.255

! NAT: внутренние адреса заменяются на адрес внешнего интерфейса (PAT)
ip nat inside source list 1 interface GigabitEthernet0/5 overload

! Пометить внутренние интерфейсы
interface GigabitEthernet0/0
 ip nat inside
 exit
interface GigabitEthernet0/1
 ip nat inside
 exit
interface GigabitEthernet0/2
 ip nat inside
 exit
interface GigabitEthernet0/3
 ip nat inside
 exit
interface GigabitEthernet0/4
 ip nat inside
 exit

! Внешний интерфейс
interface GigabitEthernet0/5
 ip nat outside
 exit

exit
write memory
```

Проверка:

```
show ip nat translations
show ip nat statistics
```

## 8. Коммутаторы (Switches)

- Обычно достаточно подключить устройства к портам и при необходимости задать hostname.
- VLAN не обязательны, если по заданию не требуются.

Минимально на каждом коммутаторе:

```
enable
configure terminal
hostname SW1
exit
write memory
```

---

## 9. Хабы (Hubs)

Хабы в Packet Tracer не настраиваются (уровень 1). Достаточно подключить кабели к портам.

---

## 10. Клиентские ПК

- **IP:** либо **DHCP** (тогда в настройках интерфейса выберите DHCP), либо статический из нужной подсети (E, B или C).
- **Gateway:** первый IP подсети (адрес маршрутизатора), например 68.76.115.1 для подсети E.
- **DNS:** 68.76.115.130 (IP DNS-сервера).
